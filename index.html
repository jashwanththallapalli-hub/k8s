Module 1

Introduction to Jenkins

Jenkins is an open-source automation server used to continuously build, test, and deploy software. In CI/CD workflows, Jenkins orchestrates repetitive engineering tasks so teams can release reliably and faster.

Why Jenkins in CI/CD?

Continuous Integration

Automatically runs builds and tests for each code push, reducing integration failures and feedback time.

Continuous Delivery

Packages and verifies software artifacts so releases remain repeatable and low risk.

Extensible ecosystem

2000+ plugins connect Jenkins to GitHub, Kubernetes, Slack, Docker, SonarQube, Terraform, and cloud platforms.

  • Controller: schedules jobs, stores configs, manages credentials and plugins.
  • Agents: execute builds in parallel and isolate toolchains.
  • Executors: worker slots on each node deciding concurrent jobs.
