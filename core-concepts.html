Module 3

Jenkins Core Concepts

Master jobs, builds, nodes, agents, executors, workspaces, plugins, labels, and scheduling before diving into advanced automation.

Jobs and Builds

A job defines automation steps. A build is a specific run instance with logs, artifacts, and status.

Nodes, Agents, Executors

Nodes are machines connected to Jenkins. Agents execute workloads. Executors define how many builds can run simultaneously per node.

Workspaces

Each job gets a workspace directory where source code is checked out and build commands execute.

Plugins

Plugins add SCM integrations, notifications, deployment tools, and security enhancements.

Labels and Scheduling

Use labels like linux && docker to route jobs to compatible agents. Use cron triggers for timed jobs.

Task

Create two agents (linux, windows) and schedule a nightly backup job on linux label.
