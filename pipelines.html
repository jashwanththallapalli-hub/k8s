Module 4

Jenkins Pipelines

Pipelines implement CI/CD as code using Jenkinsfile. Stages are transparent, version-controlled, and auditable.

Declarative Pipeline

Opinionated syntax, easier for teams, built-in post actions and agent declarations.

Scripted Pipeline

Groovy-based flexible approach for complex logic and dynamic flow.

Jenkinsfile Example (Build-Test-Deploy)

pipeline {
  agent any
  stages {
    stage('Build') { steps { sh 'npm ci && npm run build' } }
    stage('Test')  { steps { sh 'npm test -- --ci' } }
    stage('Docker') { steps { sh 'docker build -t app:${BUILD_NUMBER} .' } }
    stage('Deploy') { when { branch 'main' } steps { sh './deploy.sh' } }
  }
  post {
    always { archiveArtifacts artifacts: 'dist/**', fingerprint: true }
    failure { mail to: 'devops@company.com', subject: 'Pipeline Failed', body: "Check ${env.BUILD_URL}" }
  }
}
